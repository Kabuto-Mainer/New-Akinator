# GKN — GraphicKabutoNovel

GKN — это движок для создания визуальных новелл. Достаточно описать сюжет и подготовить текстуры интерфейса — всё остальное берёт на себя программа.

## Структура проекта

Настройка выполняется через файлы в двух директориях:

* `Configs/` — параметры объектов на экране (геометрия, привязка текстур, действия). Сейчас редактирование этой директории не отлажено, поэтому изменять её содержимое следует осторожно.
* `Data/` — содержимое новеллы: текстуры и сюжет.
  * `Data/Img/` — все изображения, на которые ссылаются конфиги и сценарий.
  * `Data/Base/1.conf` - текущий сюжет новеллы

## Этапы настройки

1. Настройка меню *(полноценная настройка пока не поддерживается)*.
2. Настройка объектов — на текущий момент поддерживается только редактирование текстур и текста внутри объектов.
3. Запись сюжета.

## Настройка объектов

Объекты описываются в `Configs/obj.conf`. Доступные для редактирования поля — путь к текстуре (`tex_p` и `tex_u` для кнопок, `tex` для всего остального) и текст (`data`).

```
button {
    rect    590 840 420 120
    tex_p   "Data/Img/but_pr.png"
    tex_u   "Data/Img/but_un.png"
    data    "Продолжить"
    action  4
}
```

## Сюжет

Сюжет описывается набором узлов. Каждый узел имеет имя и тело в фигурных скобках, в котором задаётся логика.

### Синтаксис

* **Имя узла** — первое слово перед `{`.
* **`step { ... }`** — отдельный этап внутри узла. Переход между этапами происходит по нажатию кнопки `Continue`. Внутри `step` можно указать:
  * `text: "..."` — текст, выводимый на экране;
  * `img:  "..."` — изображение *(опционально)*.
* **`next: "имя_узла"`** — безусловный переход к следующему узлу.
* **`choice { ... }`** — блок выбора. Если узел заканчивается блоками `choice`, в нём не должно быть `next`. Каждый `choice` описывает один вариант ответа:
  * `text: "..."` — текст варианта;
  * `go:   "имя_узла"` — узел, к которому ведёт этот вариант.

### Обязательные узлы

В сюжете обязательно должны присутствовать два узла:

* `start` — начало сюжета;
* `end` — конец сюжета.

### Пример

```
8_hw_normal_3_good {
    step {
        text: "Верно."
    }
    next: "8_hw_normal_end"
}

8_hw_normal_3_bad {
    step {
        text: "Неверно."
    }
    choice {
        text: "Закончить"
        go:   "8_hw_normal_end"
    }
    choice {
        text: "Снова"
        go:   "8_hw_normal_3"
    }
}
```
